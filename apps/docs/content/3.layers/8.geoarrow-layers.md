## Why GeoArrow

[GeoArrow](https://geoarrow.org) is a memory layout for geospatial data built on top of [Apache Arrow](https://arrow.apache.org). Compared to GeoJSON:

- **No JSON parse pass.** Coordinates live in flat typed arrays â€” the bytes you fetch are the bytes the GPU consumes.
- **Streamable.** Arrow IPC and GeoParquet are columnar and chunked; you can render a single `RecordBatch` while more arrive over the network.
- **Smaller wire size.** Float64 coordinates packed at full precision typically beat GeoJSON by 3â€“10Ă—.
- **Typed.** Each layer wrapper auto-detects the GeoArrow extension column (`geoarrow.point`, `geoarrow.linestring`, `geoarrow.polygon`, etc.) so you rarely need a manual `getPosition` accessor.

The four wrappers below bridge [`@geoarrow/deck.gl-geoarrow`](https://github.com/geoarrow/deck.gl-geoarrow) into v-maplibre's deck.gl overlay.

## Installation

```bash
pnpm add @deck.gl/core @deck.gl/mapbox @deck.gl/layers \
  @geoarrow/deck.gl-geoarrow \
  apache-arrow \
  @math.gl/polygon
```

`@math.gl/polygon` is only required for the polygon wrappers (`Polygon` + `SolidPolygon`); the scatterplot and path wrappers do not need it.

## VLayerDeckglGeoArrowScatterplot

Renders points from a GeoArrow `point` or `multipoint` column.

```vue
<script setup lang="ts">
  import { VMap, VLayerDeckglGeoArrowScatterplot } from '@geoql/v-maplibre';
  import { tableFromIPC } from 'apache-arrow';
  import type { RecordBatch } from 'apache-arrow';

  const batch = shallowRef<RecordBatch | null>(null);

  onMounted(async () => {
    const buffer = await (await fetch('/points.arrows')).arrayBuffer();
    batch.value = tableFromIPC(new Uint8Array(buffer)).batches[0];
  });
</script>

<template>
  <VMap :options="{ style, center: [-122, 37], zoom: 10 }">
    <VLayerDeckglGeoArrowScatterplot
      v-if="batch"
      id="points"
      :data="batch"
      :get-fill-color="[100, 150, 220, 200]"
      :get-radius="50"
      radius-units="meters"
      pickable
    />
  </VMap>
</template>
```

## VLayerDeckglGeoArrowPath

Renders polylines from a GeoArrow `linestring` or `multilinestring` column. Useful for GPS traces, route geometries, road networks.

```vue
<VLayerDeckglGeoArrowPath
  id="routes"
  :data="batch"
  :get-color="[255, 140, 0, 255]"
  :get-width="3"
  width-units="pixels"
  pickable
/>
```

## VLayerDeckglGeoArrowPolygon

Composite layer that internally renders both `SolidPolygonLayer` (fill) and `PathLayer` (outline). Consumes a GeoArrow `polygon` or `multipolygon` column.

```vue
<VLayerDeckglGeoArrowPolygon
  id="countries"
  :data="batch"
  :get-fill-color="[100, 150, 220, 180]"
  :get-line-color="[180, 220, 255, 255]"
  :line-width-min-pixels="1"
  stroked
  filled
  pickable
/>
```

## VLayerDeckglGeoArrowSolidPolygon

3D extruded polygons. Best for building footprints, height-mapped choropleths, voxel-style visualizations.

```vue
<VLayerDeckglGeoArrowSolidPolygon
  id="buildings"
  :data="batch"
  :get-fill-color="[200, 100, 50, 200]"
  :get-elevation="(d) => d.height_m"
  extruded
  filled
  pickable
/>
```

## Loading data

### From Arrow IPC (`.arrows`, `.arrow`, `.feather`)

```ts
import { tableFromIPC } from 'apache-arrow';

const buffer = await (await fetch(url)).arrayBuffer();
const table = tableFromIPC(new Uint8Array(buffer));
const batch = table.batches[0]; // pass this to :data
```

### From GeoParquet

GeoParquet decoding is not part of `apache-arrow` itself. Use [`parquet-wasm`](https://kylebarron.dev/parquet-wasm/) or [`@loaders.gl/parquet`](https://loaders.gl/docs/modules/parquet) to convert a `.parquet` file into an Arrow `Table`:

```ts
import { readParquet } from 'parquet-wasm';
import { tableFromIPC } from 'apache-arrow';

const wasmTable = readParquet(new Uint8Array(buffer));
const table = tableFromIPC(wasmTable.intoIPCStream());
const batch = table.batches[0];
```

## Sample data

The official [`geoarrow/geoarrow-data`](https://github.com/geoarrow/geoarrow-data) repository hosts CORS-enabled sample files you can use to prototype:

| File | Size | Geometry |
| --- | --- | --- |
| `natural-earth_countries-geography.arrows` | 178 KB | Polygon (countries) |
| `natural-earth_cities.arrows` | ~70 KB | Point (cities) |
| `microsoft-buildings_*` | varies | Polygon (buildings) |

Stream them via jsDelivr's CDN for snappy reloads:

```ts
const url =
  'https://cdn.jsdelivr.net/gh/geoarrow/geoarrow-data@main/natural-earth/files/natural-earth_countries-geography.arrows';
```

## Picking & events

All four wrappers emit `@click` and `@hover` with a `PickingInfo` payload. For GeoArrow specifically, the `info.object` is `null` (no per-row materialization), but `info.index` and `info.sourceLayer.props` are available; use the Arrow `RecordBatch.get(index)` API to read row data on demand.

```vue
<VLayerDeckglGeoArrowPolygon
  id="countries"
  :data="batch"
  pickable
  @click="(info) => console.log('row', info.index, batch?.get(info.index))"
/>
```
